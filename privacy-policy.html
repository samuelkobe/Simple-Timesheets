Simple Timesheets

Privacy Policy

Last updated: May 2026

Simple Timesheets does not collect, transmit, or store any of your data on any server we own or operate. Ever.

What data the extension handles

Simple Timesheets handles the following data locally on your device:

  • Project names, hourly rates, and settings you configure
  • Time entries including date, duration, and optional memos you write
  • Your Google account name, email address, and profile photo (only if you choose to sign in)

Where your data lives

All data is stored in two places, both under your exclusive control:

  • Your browser (chrome.storage.local) — the primary data store. It lives on your device, inside Chrome, and is not accessible to us or any third party.
  • Your Google Drive (optional) — if you choose to enable Drive sync, your data is saved to a private, hidden folder in your own Google Drive account. This folder is only accessible to Simple Timesheets when you are actively signed in, and is invisible in the Google Drive UI. We do not have access to it. Google's own privacy policy governs your Drive storage.

What we do not do

  • We do not run any servers that receive your data
  • We do not use analytics, tracking pixels, or telemetry of any kind
  • We do not sell, share, or monetize your data
  • We do not have access to your Google Drive or its contents
  • We do not store your Google account credentials — authentication is handled entirely by Google's OAuth service

Google Sign-In (optional)

Signing in with Google is entirely optional. It is only used to enable Drive sync. When you sign in, Google provides the extension with an access token to write to your Drive's Application Data folder. This token is never transmitted to us. You can revoke access at any time from your Google Account permissions page.

Google API Services — Data Access Disclosure

Simple Timesheets' use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

The following table documents exactly how each Google API scope is used:

Scope Data accessed How it is used Stored / shared
drive.appdata A single JSON file in the user's private Google Drive Application Data folder (invisible to the user in Drive UI, inaccessible to any other app) Read on startup to restore the user's timesheet data; written when data changes to keep it backed up. Used solely to sync data across the user's own devices. Stored only in the user's own Google Drive. Never transmitted to any developer-operated server. Never shared with any third party.
userinfo.profile Google account display name and profile photo URL Displayed inside the extension's Settings panel so the user can confirm which Google account is connected. Not processed or analysed in any way. Held in memory only for the current session. Never written to any server or persistent storage outside the user's own device.
userinfo.email Google account email address Displayed alongside the profile name in the Settings panel so the user can confirm which account is linked to Drive sync. Not processed or analysed in any way. Held in memory only for the current session. Never written to any server or persistent storage outside the user's own device.

Google user data is never used for advertising, profiling, or any purpose beyond the specific feature described above. The extension does not transfer Google user data to third parties. The extension does not use Google user data to train AI or machine learning models.

Data export and deletion

You own your data completely. You can export a full copy at any time using the Export JSON feature in Settings. To delete your data, use the "Clear All Data" option in Settings. If you enabled Drive sync, you can also delete the app data folder from your Google Account at any time.

Children's privacy

Simple Timesheets is not directed at children under 13. We do not knowingly collect any information from children.

Changes to this policy

If we make material changes to this policy, we will update the "Last updated" date above. Because the extension stores no data on our side, changes to this policy will only ever reflect changes in how the extension itself behaves — not changes to a backend.

Open source

Simple Timesheets is open source. You can read every line of code to verify these claims yourself.
